# 参与万轴

贡献分两类：**数据**和**代码**，走不同的路径。

## 一、数据贡献（补书、纠错、加书）

不需要技术背景，直接在站点上操作：

- **补全 / 纠错**：在图书或作者页面点「补全 / 纠错」，修改字段后提交。
- **添加图书**：在「添加图书」输入 ISBN，系统会尽量预填，确认后提交。
- 所有提交进入待审队列，经人工审核后入库。匿名即可参与，注册账号可追踪自己提交的处理进度。

**批量提交**：把书目按 CSV 模板整理好，邮件发送至站点公布的邮箱，由管理员导入为待审条目。模板与字段说明见站点「联系」页。

数据贡献请勿走 GitHub。

## 二、代码贡献

环境：Python 3.12+，包管理用 [uv](https://docs.astral.sh/uv/)。

```bash
git clone <repo>
cd <repo>
uv sync
uv run pytest                 # 跑测试
CNBIB_DB=data/library.db uv run uvicorn cnbib.api:app --reload   # 本地起服务
```

流程：

1. Fork 并新建分支。
2. 改动尽量聚焦一件事。
3. 核心逻辑配单元测试，保持 `uv run pytest` 全绿。
4. 提交 Pull Request，说明改了什么、为什么。

约定：

- 代码风格与改动量向周围现有代码看齐。
- 不要提交数据库文件、密钥或其它机密（已在 `.gitignore` 中）。
- 较大的改动（数据模型、合并优先级、审核规则等）建议先开 Issue 讨论再动手。

## 反馈

Bug 与建议开 Issue；代码改进提 PR。
